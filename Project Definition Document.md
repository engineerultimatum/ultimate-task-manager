## 1. Project Name

**UTiM (Ultimate Task Manager)**

---

## 2. Project Summary

A task manager application that allows users to plan their daily activities, manage their tasks, track their progress, and stay motivated.

---

## 3. Objectives

- Engaging UI
- A reward or progression system that visualizes user's achievements
- Basic functions of a task manager app such as charts and TO DO lists

---

## 4. Scope

This project is for task management which naturally includes time management but it is not a calendar app. It rewards the users using progression and achievements but it is not a simple game to collect points. It is not a detailed project manager.

---

## 5. Target Audience

General audience, mostly students looking for effective ways to organize their plan.

---

## 6. Key Features

- Add task function
- Delete task function
- Complete task function
- Update task function
- Reorganizing task schedules function

---

## 7. Deliverables

- Multi-platform task management application (Desktop, Web)
- User login system with seed-based save management
- Hierarchical task organization (tasks with nested subtasks)
- Task creation, editing, deletion, and completion functionality
- Task importance levels (Low/Medium/High) with color-coded visualization
- Point-based reward system that tracks user achievements
- Points awarded upon task completion based on importance level
- Calendar view with monthly navigation
- Deadline scheduling for tasks
- Time-remaining countdown display for deadline-based tasks
- Visual deadline indicators in task list

---

## 8. Budget and Resources

This project team consists of 5 team members all of which will contribute accordingly. Cloud services may be needed to hold user account data. Learning resources will generally be online sources.

---

## 9. Risks and Mitigation Strategies

**Primary Risks:**
- Disorganization of team members
- Secondary schedules of members conflicting with project requirements
- Lack of technical experience

**Mitigation Strategy:** Excellent organization of the team is necessary to address these risks.

---

## 10. Project Success Criteria

**Main Success Criteria:**
- Completion of basic task management requirements: task adding, task deleting, and task completing

**Secondary Success Criteria:**
- Reward system implementation
- More in-depth task management functions
