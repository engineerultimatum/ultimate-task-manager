## 1. Project Scope

This project is for task management which naturally includes time management but it is not a calendar app. It rewards the users using progression and achievements but it is not a simple game to collect points. It is not a detailed project manager.

**Includes:** Task creation, editing, completion, hierarchical subtasks, deadline scheduling, reward system, multi-language support (English, Turkish), desktop and web platforms.

**Excludes:** Team collaboration, cloud sync, detailed resource management, gamification as primary feature.

---

## 2. Project Organization

Project will be organized by tasks decided and divided between team members in weekly meetings.

---

## 3. Objectives

- Engaging UI with smooth animations and intuitive navigation
- A reward or progression system that visualizes user's achievements
- Basic functions of a task manager app such as task lists, hierarchical organization, and deadlines

---

## 4. Key Phases and Timeline

- **Week 1:** Requirements gathering and design
- **Week 2-3:** Core development (login, task management, data storage)
- **Week 3-4:** UI implementation and styling
- **Week 5:** Feature integration (progression system, calendar)
- **Week 6:** Testing and refinement
- **Week 7:** Deployment and project closure

**Expected Completion Date:** End of Week 6  
**Project Evaluation Date:** Week 7

---

## 5. Resource Planning

**Tools & Technologies:**
- Rust with Dioxus 0.7.1 framework
- Tailwind CSS for styling
- serde_json for data persistence
- Cargo and dx CLI for development
- Git for version control

**Access:** All tools are open-source and freely available. No licensing costs.

---

## 6. Risk Management

Main tool for risk management is regular communication between team members to ensure everything is on track.

**Potential Risks:**
- Cross-platform compatibility issues → Regular testing during development
- Performance bottlenecks → Early profiling and optimization
- Scope creep → Strict adherence to project scope definition
- Team member unavailability → Clear documentation and knowledge sharing

---

## 7. Communication Plan

Communication will happen at anytime it is needed, generally through online channels and with in person meetings when deemed necessary.

- Weekly planning and sync meetings
- Online communication for daily updates
- In-person meetings for critical decisions

---

## 8. Change Management Plan

Changes will be managed by first determining its scope and amount. Updated tasks will be redistributed between team members through online channels or in person meetings if necessary.

- **Low-impact changes** (< 4 hours): Approve and redistribute at next meeting
- **High-impact changes** (> 4 hours): Submit for team review before proceeding

---

## 9. Budget Plan

Budget problems aren't expected at this time. Biggest constraint is expected to be time.

**Estimated total effort:** ~100 hours
